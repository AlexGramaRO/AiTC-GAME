1. Overview

AiTC is a browser-based air traffic control training simulator. You prepare sectors (waypoints, routes, boundaries) and exercises (traffic scenarios), then run a simulation with a radar-style display, aircraft labels, and data-link style commands.

This manual describes what you see on screen, which buttons to use, and how settings behave. The built-in default settings are assumed unless you have changed them (for example in SIM Settings or by loading a saved preset).

2. Home & entry

2.1 Main screen

Figure — Home buttons (conceptual)
▶ Start Simulation ⚙ Settings
  • Start Simulation — Opens exercise selection, then the Run type choice (Host / Join / Single user).
  • Settings — Opens the Settings modal for airspace, exercise, and aircraft tools.

3. Settings hub

The modal shows buttons grouped as:

  • Create Airspace, Edit Airspace, Delete Airspace — Airspace geometry, waypoints, routes, map layers.
  • Create Exercise, Edit Exercise, Delete Exercise — Links an exercise to a sector and defines traffic.
  • Edit Aircraft database — Overrides performance data; optional Turn Radius editor.

Your changes are saved in the application and are available the next time you open Settings or start a run.

3a.6 Edit Airspace (full-screen sector editor)

Top strip — buttons (all left-click)

  • Airspace dropdown — select which sector you are editing.
  • Add Waypoint — opens add-waypoint modal (coordinates can be typed or taken from map placement flows).
  • Add Multiple Waypoints — bulk import dialog.
  • Add Route — opens add-route dialog.
  • Add Multiple Routes — bulk routes import.
  • ATC Sectors — opens ATC sectors manager (boundaries + vertical limits per sector).
  • Add Airports — opens airports / runways / approaches editor.
  • Save Airspace — persists current sector data.
  • Duplicate Airspace — opens name dialog; Create / Cancel are left-click.
  • BG opacity slider — drag with left mouse to change stacked map-background opacity.
  • + / − zoom buttons — zoom in/out (left-click).
  • Reset view (toolbar, house icon) — fit/center map on the airspace (left-click).
  • # — toggle grid overlay (left-click).

Map area (sectorCanvas)

  • Pan: Left-drag on the map (when not in a placement/delete/edit/picking mode that uses the crosshair).
  • Zoom: Mouse wheel over the map (zoom is centred on the map centre / CENTRE waypoint logic in code).
  • Measure distance: Middle mouse button down on the map → drag → release to finish; a coloured line and label appear. Left-click a measurement’s text label to remove that measurement.
  • Context menu (waypoints, routes, ATC, backgrounds): Right-click on the visualization area around the map while Edit Airspace is open and not already inside a special map mode (placement, delete waypoint, edit waypoint, add-route picking, route delete/edit, add/delete ATC sector). A custom menu opens; hover submenus (Waypoints, Routes, ATC Sector, Background) then left-click a command. Commands start modes or open modals as follows:
    • Add Waypoint: cursor crosshair; left-click the map to drop a point → opens Add Waypoint with coordinates filled.
    • Delete Waypoint: crosshair; left-click a waypoint to delete (with confirmation).
    • Edit Waypoint: crosshair; left-click a waypoint → opens edit form.
    • Add Route: crosshair; left-click waypoints in order to build the chain; middle-click a waypoint removes the last occurrence of that waypoint from the chain; right-click opens a small menu to save or cancel the route.
    • Delete Route: crosshair; hover route segments; left-click to pick a route to delete (disambiguation panel if needed).
    • Edit Route: crosshair; click-drag on map to edit geometry per app rules.
    • Add ATC Sector / Delete ATC Sector / Edit ATC Sectors: enter the corresponding polygon pick or list flows; use left-click on the map or dialogs as prompted; right-click may open a cancel/context menu while a mode is active.
    • ADD / Delete / Edit Background: opens pick lists or editors; list items are chosen with left-click (see on-screen hints in those modals).

Header

  • × — close Edit Exercise (left-click).

  • Airspace / Exercise dropdowns — left-click to open; left-click option.

Sidebar — actions (left-click unless noted)

  • Add Flight — opens Add/Edit Flight modal (enabled when an exercise is selected).
  • Add Multiple Flights — opens bulk paste/import modal.
  • Save Exercise — saves flights for the current exercise.
  • Preview time slider — drag with left mouse to scrub preview time; clock updates.

Flights section (collapsible)

  • Flights (n) header row — left-click the toggle (chevron + title) to expand or collapse the list.
  • all flights listed — opens a read-only text modal with one line per flight; Copy / Close are left-click.
  • Export — downloads a CSV of flight plans (left-click).
  • Each flight row: left-click the row to select it. Buttons on the row (e.g. Edit / Duplicate / Delete) — left-click (Delete may confirm).

Exercise editor map (exerciseEditorCanvas)

  • Pan: left-drag on empty map.
  • Zoom: mouse wheel (zoom keeps the point under the cursor).
  • Drag preview label: uses the same Label move setting as the simulation (SIM Settings): left, middle, or right button on the label box, then drag. Default in bundled preset is often middle.
  • Adjust spawn time along route (preview): with preview time > 0, middle-click on an aircraft symbol (not on the label box) and drag; release to commit spawn time from along-track position.
  • Edit flight from map: right-click an aircraft symbol → opens the flight in Edit Flight modal.
  • Pick route on map (from Add/Edit Flight): when that mode is active, the Add Flight modal closes and a banner appears. Use the middle mouse button on the map: middle-click a waypoint to add or toggle it in the route sequence; middle-click empty airspace to insert a coordinate token; if several routes pass under the cursor, a disambiguation panel can open (choose with the mouse). Right-click on the map opens a context menu (finish segment, cancel, etc.). When not in pick mode, right-click an aircraft symbol still opens Edit Flight.

Add Flight / Edit Flight modal

  • × / Cancel — close (left-click). Backdrop click does not close this modal.
  • Pick on Map — starts route picking on the exercise map (left-click).
  • Create / Save (submit) — saves flight (left-click).
  • Autocomplete suggestions for aircraft type and route — left-click a suggestion to choose.

Add Multiple Flights modal

  • × / Cancel — close (left-click). Backdrop does not close.
  • Import (submit) — parses pasted lines (left-click).

3a.8 Edit Aircraft database

  • × / Close — close modal (left-click).
  • Aircraft type dropdown — left-click to choose a type.
  • Turn Radius — opens in-place turn-radius band editor; Default / Save / Close (left-click). Grid cells edited with mouse/keyboard per field.
  • Update override — writes current numeric fields into the override text for that type (left-click).
  • Remove override — removes override for selected type (left-click).
  • Overrides textarea — click to focus; edit text with keyboard.
  • Restore defaults — clears overrides in storage after confirm (left-click).
  • Save — persists overrides (left-click).

4. Airspace & exercises

For a full list of Settings-related buttons and mouse actions (Edit Airspace map, Edit Exercise map, modals), see section 3a.

4.1 Sector editor

Waypoints use latitude and longitude; a CENTRE / CENTER waypoint anchors the radar. Routes connect waypoints; boundaries define sectors.

4.2 Exercise editor

Defines which sector and which flights or aircraft appear. RUN Simulation on a card starts the run flow.

4.3 ATC sectors, airports, runways, approaches

Additional screens let you define ATC sector polygons (vertical limits), airport reference points, runways, and approaches — used by the simulation for approaches and runway alignment.

5. Aircraft database

Default performance comes from the built-in database. Overrides are per aircraft type. Turn radius bands (by flight level) affect displayed trajectories and turn behaviour. Button-by-button behaviour from Settings is in section 3a.8.

6. Starting a simulation

  1. Select an exercise, then choose RUN Simulation.
  2. Pick Host (name your session), Join (pick a hosted session), or Single user.
  3. As host, you create the session; joiners connect later. As joiner, you pick role and sector when prompted.
  4. The simulation view opens full screen with the radar canvas.
Host tab: If you close the browser tab while hosting, the application tries to end the session so it does not stay listed forever. Idle hosted sessions may also be removed automatically if the host stops sending updates.

7. Multi-station operation (host / join)

  • Host — Runs the simulation clock and aircraft movement; other participants receive updates from the host.
  • Joiners — Choose roles such as EXEC, planner, instructor, evaluator, observer, or pseudo-pilot where offered. Some roles are exclusive and may require claiming a slot.
  • Pseudo-pilot — Sends lateral (and related) commands to the host when that mode is active.

9. Radar & map

9.1 Pan & zoom

  • Scroll wheel — Zoom centred on the view (strength is set under SIM Settings).
  • Left-drag on empty map (when not using that button for label drag) — Pan the radar.

9.2 Aircraft

Aircraft appear as symbols with labels. Trajectory lines and STCA overlays depend on your SIM Settings.

9.3 Measuring

Right-drag from a point or an aircraft to measure distance (see section 10).

10. Mouse & label interactions

With the default settings, Label move is set to the middle mouse button: drag labels with the middle button. The left button is used for panning, opening pickers, and map actions; the right button is used for measure, direct-to path, and speed cycling as described below. If you change Label move in SIM Settings, use the button you chose for dragging labels instead of middle.

11. Label layout & fields

The following describes the default label layout (five rows, five columns). You can change the grid under SIM Settings → Edit Label (password may be required). If you load a different preset or saved layout, field positions may differ.

11.1 Options that go with the default layout

  • Label move: Middle button (drag the label with the middle mouse button).
  • Heading select increment: 5° steps when confirming a heading from the HDG/DCT line.
  • Probing level: Same route (exit-level probing highlights aircraft on the same route).
  • TTS: On by default in the bundled settings.
  • Trajectory display: Turn-radius style.

11.5 Editing the layout

SIM Settings → Edit Label opens the same kind of 5×5 grid so you can assign which data appears in each cell. To restore the factory layout and related options, use SIM Settings → Load preset and pick Default_Settings if it appears in the list.

Assigned level, exit level, IAS/MACH, direct waypoint list, and runway pickers open as list overlays — scroll inside the list when there are many options.

12. SIM Settings options

  • Radar update (seconds) — How often positions update and labels align with radar sweeps.
  • History dots — Up to seven past positions shown as faint dots.
  • Trajectory display — Point-to-point versus turn-radius drawing style.
  • Trajectory display of labels — Show estimates only, or estimates plus flight-level labels.
  • Label move — Which mouse button drags labels.
  • Heading select increment — 1°, 5°, or 10° when confirming a heading.
  • Zoom sensitivity — Wheel zoom strength (1–10).
  • Min LOC approach distance (NM) — Distance to the runway for arming ILS, localizer, or GPS approaches.
  • Probing level (exit level) — Off, on, same route, or same crossing routes, plus highlight and dim colours.
  • STCA — Conflict check type, separation distance, lookahead time, optional trigger dots.
  • TTS — Also available from the strip; must be enabled in settings for readbacks to work.
  • Presets — Load or save the full radar, map, and appearance settings. The application ships with a Default_Settings preset that matches the default label grid, middle-button label move, turn-radius trajectories, and probing colours described in section 11 (TTS and other options follow the values stored in that preset).

13. Operation modes

Modes include data link (DL), combinations with the planner (PLN), executive plus pseudo-pilot (EXE+PP), and other variants. Press MODE to open the selector. Hosted sessions run in the data-link style with role-specific behaviour for joiners (for example pseudo-pilot lateral commands where allowed).

14. Tips

  • Direct path (DL): Right-click the DCT field → path mode. Left-click the map or waypoints to add legs. Right-click again to cancel or finish as prompted.
  • Heading: Middle-click HDG/DCT → left-click the map for direction (step size from SIM Settings) → right-click to cancel.
  • Host away from keyboard: The simulation may pause when you hide the host tab, depending on your environment.
